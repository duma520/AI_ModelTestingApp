## 1. 软件概述
### 1.1 软件简介
本工具是一个图形化界面的深度学习模型验证与测试工具，支持多种主流深度学习模型和推理框架，旨在帮助开发者、研究人员和工程师快速验证和测试深度学习模型的性能。
### 1.2 适用人群
- **初学者**：无需编程基础，通过可视化界面操作
- **开发者**：快速验证模型效果
- **研究人员**：测试模型性能指标
- **工程师**：部署前的模型验证
### 1.3 主要功能
- 支持多种模型类型：YOLO、MobileNet、自定义模型
- 支持多种推理框架：TensorRT、ONNX、OpenVINO、原生框架
- 多种输入类型：图像、视频、摄像头、目录批量处理
- 模型验证、测试、导出和性能测试功能

## 2. 安装指南
### 2.1 系统要求
- 操作系统：Windows 10/11, Linux, macOS
- Python版本：3.6及以上
- 硬件要求：
  - CPU：Intel i5及以上
  - GPU：NVIDIA显卡（推荐，用于加速推理）
  - 内存：8GB以上
  - 存储空间：1GB以上可用空间
### 2.2 安装步骤
1. 安装Python 3.6+
2. 安装依赖库：
   ```
   pip install tkinter pillow numpy opencv-python
   ```
3. 下载本软件AI_ModelTestingApp.py文件
4. 运行程序：
   ```
   python AI_ModelTestingApp.py
   ```

## 4. 功能详解
### 4.1 模型验证
**作用**：检查模型是否可用
**使用场景**：拿到新模型后首先验证
**参数说明**：
- 模型类型：选择模型架构
- 模型路径：选择模型文件(.pt, .onnx等)
### 4.2 模型测试
**作用**：实际运行模型查看效果
**使用场景**：验证模型在实际数据上的表现
**参数说明**：
- 输入类型：选择测试数据形式
- 输入路径：选择测试文件/目录
### 4.3 模型导出
**作用**：转换模型格式
**使用场景**：准备部署模型到不同平台
**参数说明**：
- 推理框架：选择目标部署框架
- FP16/INT8：启用量化减小模型大小
### 4.4 性能测试
**作用**：测量模型运行效率
**使用场景**：优化模型性能时
**输出指标**：
- FPS：每秒处理帧数
- 延迟：单次推理耗时
- 显存占用：GPU内存使用量

## 5. 使用教程
### 5.1 基础使用流程
1. 选择模型类型和文件
2. 配置输入源（图片/视频等）
3. 点击"测试模型"查看效果
4. 根据需要调整参数重新测试
### 5.2 实例演示
**案例1：测试YOLO模型**
1. 模型类型选择"YOLO"
2. 浏览选择.pt模型文件
3. 选择配套的.cfg配置文件和.names类名文件
4. 选择测试图片
5. 点击"测试模型"
**案例2：导出ONNX模型**
1. 选择原始模型文件
2. 推理框架选择"ONNX"
3. 点击"导出模型"
4. 等待完成提示

## 6. 常见问题
### Q1: 运行时提示缺少模块
A: 使用pip安装缺失的包，例如：
```
pip install pillow
```
### Q2: 模型加载失败
A: 检查：
1. 模型路径是否正确
2. 模型类型是否匹配
3. 模型文件是否完整
### Q3: 摄像头无法使用
A: 检查：
1. 摄像头是否连接
2. 索引号是否正确（0通常是默认摄像头）
3. 是否有其他程序占用摄像头

## 7. 高级配置
### 7.1 专业参数说明
- **输入尺寸**：必须与模型训练尺寸一致
- **置信度阈值**：过滤低置信度检测结果(0-1)
- **NMS阈值**：控制重叠框合并程度(0-1)
- **批处理大小**：同时处理的样本数（影响内存使用）
### 7.2 性能优化建议
1. 启用FP16可提升速度但可能降低精度
2. INT8量化可大幅减小模型但需要校准
3. 适当增大批处理大小提高吞吐量
